049 · Same-Rect Element Disambiguation

When the user clicks a spot where multiple elements share the same bounding rect, a small picker popup lets them choose which element to select. This replaces the old behavior of always walking up to the component root, which caused wrong classes to be sent to the panel.

The algorithm: walk up from e.target, compare getBoundingClientRect() against the clicked element. Parents within ~2px on all sides are candidates. If only 1 candidate, select directly. If 2+, show the picker.

Scenario 1: No Overlap — Direct Select

The indigo banner's parent (<div class="px-6 py-5">) has padding, so its bounding rect is larger. Only 1 candidate → select immediately, no popup. The panel receives bg-indigo-600 classes (not the parent's).
