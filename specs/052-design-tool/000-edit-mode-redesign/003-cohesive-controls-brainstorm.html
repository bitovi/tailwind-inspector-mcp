Goals: no drag handle (drag from outline), no delete button (press ⌫), hide msg row during text edit, accurate pulsing outline (teal↔orange, 2px border-radius, slight offset), text + queue feel connected. Exploring layouts where everything below the element is one cohesive strip.

Layout 1 — Unified Bar (T + msg → T + queue)

One dark bar below the element. At rest: T button + message input. Click T: bar transforms to T (active) + Queue + Cancel. Everything lives in the same strip — nothing jumps around.

Key Decisions
Outline — Keep the full pulsing border (teal↔orange). It's a strong brand signal and gives the selection a quality feel. border-radius: 2px, offset: ~3px.
Msg row hides during text edit — The bar transforms in-place to show queue/cancel. Same position, same shape, different content.
T button location — Layouts 1, 2, 5 all keep T adjacent to queue. Layout 5 is most minimal (T is part of the msg row itself).
Do we need the T button? — Layout 4 drops it entirely: double-click or press T. Cleanest, but less discoverable.
Full border vs brackets? — The full border is needed for drag-anywhere and the pulsing animation. Brackets lose both. Recommend keeping full border.
