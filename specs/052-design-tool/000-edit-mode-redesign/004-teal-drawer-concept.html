Interaction Flow
1. Click element — Teal pulsing outline + "stat-card" tag appears. Teal drawer slides in below with "Aa Edit text" + msg row.
2. Click "Edit text" (or double-click text) — Outline goes solid orange. "Aa Edit text" → orange "✏ Editing text" pill + Discard/Done. Text node in page becomes editable with orange highlight + "value" tag.
3. User edits text inline — Type directly in the element. Msg row stays available for voice/text instructions to the agent about the change.
4. Click "Done" — Text change is queued. Returns to State 1 (teal outline + "Aa Edit text").
5. Click "Discard" or Escape — Text reverts. Returns to State 1.
Design Notes
Outline → orange on edit — Clear visual mode change. Teal = selected. Orange = editing. Same colors as the pulse animation (teal↔orange), so they're already in the brand vocabulary.
Drawer is the single control surface — No separate floating buttons. The drawer header morphs between modes. Msg row is always present.
"value" tag on text node — Shows users which text node is being edited. Could extend to "label", "heading", "paragraph" etc.
Resize handles dim during text edit — Can't resize while editing text. Handles fade to 25% opacity.
Drawer width — Could match element width (attached feel) or be fixed width (consistent). Mock shows fixed width for now.
