043 · Component List Redesign

Evolving the compact tile row layout from spec 024 with smart ghost thumbnails, lazy loading, better naming, and an expand-to-full-size preview. The goals: fast scan, minimal resource usage, clear selection language, and easy access to variant/props editing. See spec.md for technical specifications.

1 — Thumbnail Scaling Strategy

The ghost is rendered at natural size inside a hidden <adaptive-iframe>, then transform: scale() shrinks it to fit the 56×40 thumbnail. The scale factor is adaptive: small components (Tag, Badge) don't shrink at all; large components (Card, Nav) shrink significantly. The outer .thumb clips via overflow: hidden.

Adaptive scale — no shrink needed

Small components like Badge and Tag render at natural pixel size. The 56×40 thumb area is large enough to show them without scaling. scale(1) — zero distortion.

2 — Row Actions: Customize + Insert/Replace Buttons (3 States)

Every component row has two always-visible buttons: Customize (expands preview + props) and an Insert/Replace button whose appearance depends on page context. The button has three visual states: gray (idle), teal (page has a target — one click acts), and orange (user-initiated arming — waiting for page click).

A · Gray — no page selection

No element is selected on the page. The Insert/Replace button looks gray, identical to Customize. It's available but not highlighted — nothing urgent to act on. Clicking it enters arming mode (see state C).

3 — Lazy Loading States

Thumbnails load lazily. The buttons are always visible regardless of thumbnail state. Clicking Customize or the thumbnail placeholder triggers ghost extraction and then expands to the preview+props drawer once loaded.

A · Unloaded — placeholder

Dashed placeholder. The two action buttons are still visible and functional. Clicking "Customize" or the placeholder triggers loading → then auto-expands. Clicking "Insert" arms immediately (no preview needed to select).

4 — Expand Drawer: Full-Size Preview + Props Together

Clicking Customize or the thumbnail expands a drawer below the row showing both the full-size ghost preview and the props editor at the same time. The Insert/Replace button stays in the row header in its current state (gray, teal, or orange).

A · Expanded — gray (no page selection)

User clicked Customize or thumbnail to expand. No page element selected, so the Insert button stays gray. User can tweak props, then optionally click "Insert" to enter arming mode (orange).
