044 · ReactNode Props — UX Exploration (v2)

Exploring how users populate ReactNode props (like Badge's iconLeft and children) with text/HTML or other components. Simplified design: no mode tabs, no RN badges. ReactNode fields are just text inputs that can also accept an armed component via click. The arm → click pattern reuses the existing Insert button — it serves both page insertion and prop assignment.

1 — ReactNode Field: Two States

ReactNode props render the same text input as any other prop. The difference: when a component is armed, these fields become receptive (teal glow) and accept a click to assign the armed component. Once assigned, the text input is replaced by a component chip showing the thumbnail, name, and a ✕ clear button. Clearing reverts to the text input. No mode tabs, no special badges.

2 — Full Flow: Arm → Click-to-Set

The complete interaction for assigning an Icon component to Badge's iconLeft prop. Uses the same Insert button that already exists for page insertion — it now serves both purposes. Arming clears after one assignment.

1
Expand Badge — click Customize. See the props editor with text inputs for children, iconLeft, iconRight.
2
Find and configure Icon in the component list. Expand it, set name="check", size="sm".
3
Arm the Icon — click "Insert" on the Icon row. It turns orange ("Inserting"). Badge's ReactNode fields instantly glow teal (receptive). The page also accepts clicks — same armed state serves both targets.
4
Click Badge's iconLeft field — the armed Icon is assigned. Field switches to the component chip. Badge's ghost re-renders showing the check icon. Arming clears. All other fields return to normal.
5
Done. Arm Badge itself and place it on the page. The agent receives: <Badge variant="primary" iconLeft={<Icon name="check" size="sm" />}>Active</Badge>

4 — HTML/SVG in Text Input

Since there's no separate "HTML mode", pasting HTML or SVG directly into the text input just works. The text is used as-is in the ghost (stitched into the slot) and converted to JSX for the agent. This covers inline SVG icons, <strong> wrappers, and other simple markup.

User pasted an SVG check icon directly into the iconLeft text input. The ghost preview shows the SVG rendered in the badge's left icon slot. No mode switch needed — the text input accepts anything.

5 — Decisions
✓ Arming clears after one assignment
Arm Icon → click one prop field → Icon assigned, arming cleared. Simple and predictable. To assign the same component to multiple slots, arm again.
✓ Same Insert button serves page and prop slots
One armed state. User can click the page to insert OR click a ReactNode field to assign. Wherever the click lands, that's where the component goes. No separate "Use as Prop" button.
✓ No mode tabs — text input is the default, component chip replaces it
ReactNode fields are just text inputs. HTML/SVG can be pasted directly. When a component is assigned via arming, the text input swaps to a chip. ✕ clears back to text. Zero extra UI chrome.
? Open: How does the field know a click means "assign" vs normal text editing?
When a component is armed AND the user clicks a receptive field, it assigns. When nothing is armed, clicks behave as normal (focus, cursor placement, text selection). The teal glow makes the "receptive" state visually obvious.
