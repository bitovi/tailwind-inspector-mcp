Flex Item Controls — Panel UX Prototype

Design prototypes for flex child/item property controls. These appear when the selected element lives inside a flex container. The controls cover: flex-1/auto/initial/none, self-* (align-self), grow/shrink, basis-*, and order-*.

1 Flex Shorthand — 2×2 Diagram Grid

The four mutually exclusive flex shorthand values, shown as mini diagrams that visually represent how items grow/shrink. Matches the existing FlexDirectionSelect grid pattern.

Interaction: Same as existing flex-container diagram selects.
  • Hover → preview flex-1 etc. on the element
  • Click → stage the class change
  • Active = teal border + shadow ring
  • flex-none uses a dashed border to hint "rigid"

Visual metaphors:
  • flex-1 — equal bars fill 100% (ignore content size)
  • flex-auto — different widths fill 100% (respect content)
  • flex-initial — bars are content-sized, trailing gap
  • flex-none — bars are fixed, dashed = "locked"

3 Grow / Shrink — Toggle Chips

Simple on/off toggles. These are secondary to the flex shorthand (which sets grow + shrink together), but useful for fine-grained control. Styled as compact segmented controls.

grow
shrink
When to use: These override the grow/shrink set by the flex shorthand. For example, flex-1 implies grow=1, shrink=1 — but you can add shrink-0 to prevent that item from shrinking.

Most users will just use the flex shorthand above. These are for power users.
4 Basis & Order — Scrubber Chips

Nested flex: When an element is both a flex container and a flex child, both control blocks appear:
  • Top: Container controls (direction, wrap, justify, align, gap)
  • Divider with context badge
  • Bottom: Item controls (flex shorthand, align-self, grow/shrink, basis, order)

The context badge ("inside flex container") helps the user understand that these controls affect how the element behaves within its parent.
