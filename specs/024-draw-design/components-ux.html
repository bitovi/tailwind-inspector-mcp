Components Panel — UX Explorations

Three variations for the component browser. Goals: make it dead-simple to find a component, click it, and insert it into the page. Hide customization controls by default. Eliminate the green "args" dot. Make the armed/insert state much more obvious.

A · Compact Tile Rows
Each component is a single row with a small thumbnail. Click the row to arm it for insertion. A gear icon reveals the props drawer inline. No chrome until you need it.

Design Notes
Green dot removed All three variations drop the green "supports args" dot. Props existence is discoverable via the gear icon (A/B) or "Customize" disclosure (C) — no passive indicator needed.
Props hidden by default None of the variations show Props/Args automatically. In A, clicking the gear icon opens an inline drawer. In B, the gear opens a popover. In C, "Customize props…" is a disclosure toggle. This keeps the primary flow — scan → click → insert — frictionless.
Click → armed is now obvious A: Teal border + background tint + "Armed ✓" badge text.
B: Teal outer glow (box-shadow) + status text ("✓ Armed — click page to place") overlaid on the preview.
C: Explicit "Insert into Page" button that toggles to orange "✕ Cancel Placement" — the strongest affordance.
Recommendation Variation A is the best balance for a sidebar panel — compact enough to show many components, fast to scan, and the inline props drawer is optional. A search bar at the top scales to larger projects. The "Armed ✓" pill provides clear feedback without consuming space.
Shrinking previews
(iframe scale trick)		 The thumbnails in Variation A use transform: scale() — the same technique that works with <adaptive-iframe>. The component renders at its natural size inside a wider container, then the whole container is scaled down visually. A clip wrapper hides the overflow:

/* CSS */
.thumb { width: 56px; height: 40px; overflow: hidden; position: relative; }
adaptive-iframe {
  position: absolute; top: 0; left: 0;
  width: calc(56px / 0.35);   /* render at natural width */
  transform: scale(0.35);
  transform-origin: top left;
  pointer-events: none;
}
Scale factor = thumb-width ÷ component-natural-width. A 0.35 scale works well for sidebar thumbnails; adjust per component type (buttons ~0.5, full cards ~0.3). Note: transform does not affect layout — the parent container still needs an explicit fixed size.
